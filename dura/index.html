What is DURA?

DURA is a neuroscience visualization tool optimized for the iPad & iOS 7. It's part of a shared codebase called WholeSlide Open which aims at providing high quality tools for medical education. DURA's name originates from dura mater, the fibrous layer containing your central nervous system.

What can it do now?

  • View high resolution image sets from Brainmaps.org (uses boring 2D viewer, 3d in progress)
  • Link with Dropbox (sandboxed currently, needs appstore submission before this will work)
  • Connect to Knossos (EM) image services (experimental, issues with VRAM and opengl es 2)
  • Connect to OpenConnectome (EM) servers (experimental, issues with VRAM and opengl es 2)
  • Open converted EyeWire neuron meshes (experimental, issues with VRAM and opengl es 2)

What will it do in the future?

  • Connect to Aperio image servers
  • Contain most of the image resources available in the original WholeSlide app.
  • Connect to MicroBrightField's Biolucida Server.
  • Connect to data stored on remote hard drives
  • View and create annotations on high resolution images
  • View stacks of high resolution images in 3D
  • Visualize skeletonized annotations for EM
  • Visualize volume annotations for EM
  • Include data sources beyond neuroscience (dermatology, radiology)
  • Integrate a searchable backend
  • Create courses and link content together

Electron Microscopy fun

Configuring a knossos server with authentication
  1. Tap on the "+" in the top left to open a new browser tab.
  2. Tap on the collections button
  3. Tap and hold on the 'mpimf-heidelberg'. This should bring up a detailed view.
  4. Tap "save to device", then close the info window by tapping the 'x'
  5. Navigate back to the main page (left as an exercise for the user)
  6. Tap on "This Device"
  7. You'll see the server that was saved
  8. Tap and hold to bring up the detailed view again
  9. Enter the username and password in lower left corner. Tap Login
  10. If successful, tap save one more time
  11. Now you can navigate to the Knossos volume view by either tapping on 'open' in the detailed view, or by closing the detail view and tapping on the icon original icon.
Navigating within an EM dataset
  1. To navigate around a volume, simply tap on the plane to adjust and drag along it's normal vector (aka perpendicular to the plane itself)
  2. Once you have the desired cube location, tap on the 'file' button on the top menu bar to download the cube and load.
  3. You can navigate to canonical views with the top toolbar as well.
  4. (very buggy currently, 3d view requires a rewrite)
Why is this so complicated? DURA maintains two datasources: local and remote representations. Local representations are stored in the Documents directory of the application itself. Remote representations are immutable and stored on a remote server somewhere. Therefore, if we need to view something that requires authentication, we need to make a local copy of it first before setting the username and password. I'll work on making this process more clear, but this approach provides a LOT of versatility.

Data models!

DURA has a nice data model that serializes easily to JSON. This will be helpful in the future.
Example: a single zoomify image from BrainMaps
{
    "url": "http://brainmaps.org/HBP3/c.auratus/hor/0602/21b",
    "create_date": "2014-02-12T15:29:04Z",
    "thumbnail_base": "http://brainmaps.org/HBP3/c.auratus/hor/0602/21b/TileGroup0/0-0-0.jpg",
    "native_size": "13440.000000,28740.000000",
    "modify_date": "2014-02-12T15:29:04Z",
    "tile_size": "256.000000,256.000000",
    "class_name": "wsRemoteZoomifyObject",
    "notifcation": "kPresentObject",
    "title": "6169"
}
It's possible to create a json representation, give it a .ws extension, and put it in the app documents directory (via itunes file sharing) and it will load. Again - this is just the start of a flexible vis environment
